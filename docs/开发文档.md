### 架构拆分

| 组件                 | 部署位置             | 说明                             |
| -------------------- | -------------------- | -------------------------------- |
| PaddlePaddle/OCR模型 | Docker容器           | 负责OCR识别核心计算              |
| FastAPI服务          | 本地开发环境         | 业务逻辑"Lifecycle"管理和API接口 |
| 模型文件             | 本地目录（挂载到容器 | 通过volume共享                   |

### 实现步骤

**1. 修改docker-compose.yml，仅运行Paddle服务**

```bash
services:
  paddle-ocr:
    build: .
    container_name: paddle-ocr-server
    ports:
      - "8000:8000"
    volumes:
      - ./uploads:/app/uploads
      - ./models/official_models:/root/.paddlex/models
      - ./logs:/app/logs
    environment:
      - PADDLEX_HOME=/root/.paddlex
    restart: unless-stopped
    command: ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**2. 本地开发环境配置**

```bash
APP_NAME=Document OCR (Local)
HOST=127.0.0.1
PORT=8001
UPLOAD_DIR=uploads
MODEL_DIR=models
LOG_DIR=logs
```

**3. 本地启动业务服务**

```bash
# 创建虚拟环境
python -m venv .venv
.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动本地服务（端口8001）
python start.py
```

### 数据流向

```tex
本地业务代码 (localhost:8001)
        │
        ▼
  HTTP请求转发
        │
        ▼
Docker Paddle服务 (localhost:8000)
        │
        ▼
   OCR模型推理
        │
        ▼
    返回结果
        │
        ▼
本地业务代码处理
```

### 优势

1. **隔离性**：Paddle依赖复杂环境隔离在容器中
2. **开发效率**：业务代码支持热重载，修改立即生效
3. **资源利用**：模型加载在容器中，不占用本地资源
4. **一致性**：生产和开发环境使用相同的Paddle版本

### 注意事项

- 确保本地和容器的模型目录同步（通过volume挂载）
- 本地服务如需调用OCR，需通过HTTP请求到容器的8000端口
- 首次构建Docker镜像需下载较大的PaddlePaddle依赖







### 本地开发 + Docker（高级用法）

**8000 和 8001 同时使用**：

| 端口 | 运行位置     | 用途                                 |
| :--- | :----------- | :----------------------------------- |
| 8000 | Docker容器   | 只运行 PaddleOCR 模型服务            |
| 8001 | 本地开发环境 | 运行业务代码（FastAPI + 自定义逻辑） |

**目的**：让业务代码支持热重载，修改代码立即生效，而模型服务稳定运行在容器中。


```bash
# 1. 启动 Docker 中的 PaddleOCR（8000端口）
docker compose up --build paddle-ocr

# 2. 本地启动业务服务（8001端口，支持热重载）
python start.py

# 3. 访问本地服务
http://localhost:8001
```

### 总结

- **如果你只是用 Docker 部署运行**：只用 8000 端口，不需要 8001
- **如果你需要本地开发调试**：8001 用于本地代码热重载，8000 用于容器中的模型服务

**你现在的情况**：直接用 `docker compose up --build paddle-ocr` 启动，访问 `http://localhost:8000` 即可，**不需要 8001**。
